# 視覺化戰力圖與儀表板設計文件 (Visual Analytics Design)

## 1. 目標描述
將側邊欄枯燥的數字轉化為直觀的「遊戲化」視覺組件，提升專業感與判讀效率。重點在於一眼看出標的是「全能型」、「偏科型」還是「過熱型」。

## 2. 核心設計方案

### 方案 A：五維戰力圖 (Neo Radar Chart)
- **維度自定義**:
  - **價值 (Value)**: 基於 PE/PB (越低越強)。
  - **獲利 (Profit)**: 基於 ROE (越高越強)。
  - **增長 (Growth)**: 基於 EPS/Rev Growth (越高越強)。
  - **回饋 (Return)**: 基於 Yield 殖利率 (越高越強)。
  - **安全 (Safety)**: 基於 Current/Quick Ratio (越高越強)。
- **視覺**: 半透明螢光綠填充，帶有外框發光，極簡網格。

### 方案 B：動能壓力儀 (RVOL Gauge)
- **維度**: 相對成交量 (RVOL)。
- **視覺**: 270 度圓弧規。
  - 0~1.0x: 灰色 (量縮冷卻)。
  - 1.0x~2.0x: 藍色 (溫和補量)。
  - >2.0x: 紅色發光 (爆量噴發)。

### 方案 C：子彈對比圖 (Industry Bullet Chart)
- **維度**: 殖利率與 PE。
- **對比**: 實條 (標的值) vs 虛線/陰影區 (產業平均)。

## 3. 佈局重組 (Dashboard Layout 2.0)
- **頂層**: 價格 + 漲跌幅 + **RVOL 儀表 (方案 B)**。
- **中層**: **五維戰力圖 (方案 A)** (核心視覺焦點)。
- **底層**: 產業版塊與增長數據 (純文字/子彈圖)。

## 4. 實作挑戰
- **數據標準化 (Normalization)**: 需要將不同單位的數據 (如 PE: 20, ROE: 15%) 映射到 0-100 的雷達圖坐標。
- **產業平均值**: 目前 API 可能缺少即時產業平均數據，初期將使用靜態門檻值 (Static Thresholds) 模擬。

## 5. 驗證標準
- 雷達圖形狀隨股票特性明顯改變。
- 爆量股 (RVOL > 2) 儀表盤應呈現顯眼的紅色提示。
